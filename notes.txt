pasta é apenas organização física, o que define 'camada' é quem chama quem

estar no plano essentials do cognito (customização de access tokens)

O controller recebe uma requisição e delega para a camada responsável
	- agnóstica a evento: não sabe que está sendo chamado por exemplo a partir de um API Gateway


pasta application teremos o código da aplicação, que não tem dependência ou relação com serviços externos, como framework, gateways, bancos de dados, serviços de autenticação....

pasta kernel são funcionamentos do framework: decorators, schema, registry, container di...

pasta contratos: tipagens criadas manualmente

pasta main é a porta de entrada da aplicação (por exemplo, onde sobe o servidor)

pasta functions vai ficar com as funções lambda

pasta shared são códigos compartilhados entre as camadas: usecases, kernel, main, etc.

pasta infra vai ter implementações com serviços externos: DBs, Gateways, Filas, Arquivos, etc.
